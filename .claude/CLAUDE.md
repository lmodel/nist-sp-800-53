## Project Overview

Electronic (LinkML) version of NIST SP 800-53 Rev 5 Controls and SP 800-53A Rev 5 Assessment Procedures. The schema models NIST security controls in a machine-readable format and generates multiple output formats (Python, JSON Schema, TypeScript, OWL, GraphQL, etc.).

## Commands

This project uses `just` as a command runner and `uv` as the Python package manager.

```bash
just install              # Install dependencies (uv sync --group dev)
just test                 # Run all tests (schema, Python, examples, OSCAL validation)
just lint                 # Lint the schema with linkml-lint
just gen-project          # Regenerate Python datamodel and all project artifacts
just gen-doc              # Regenerate documentation
just validate-oscal-all   # Validate all OSCAL documents against the schema
just clean                # Remove all generated files
```

Run a single pytest test file:
```bash
uv run pytest tests/test_data.py -v
```

Run pre-commit hooks manually:
```bash
uv run pre-commit run --all-files
```

## Architecture

### Schema-Driven Generation

The single source of truth is `src/nist_sp_800_53/schema/nist_sp_800_53.yaml` (LinkML YAML). Everything else is generated from it:

- `src/nist_sp_800_53/datamodel/` — Python dataclasses and Pydantic models (committed, regenerated with `just gen-project`)
- `project/` — JSON Schema, GraphQL, OWL, TypeScript, Java, Protobuf, SHACL, SHEx, SQL DDL (git-ignored, regenerated)
- `docs/elements/` — Per-class/slot markdown documentation (generated by `just gen-doc`)

**Only edit the schema YAML.** Never hand-edit files in `datamodel/` or `project/`.

### Test Structure

- `tests/test_data.py` — Parametrized pytest that validates YAML/JSON files under `tests/data/`
- `tests/data/valid/` — Files expected to pass schema validation
- `tests/data/invalid/` — Files expected to fail schema validation
- `tests/data/nist/` — Official NIST OSCAL documents (catalog, LOW/MODERATE/HIGH/PRIVACY baseline profiles, resolved catalogs)

New schema elements require both valid and invalid example data before merging (see `CONTRIBUTING.md`).

### OSCAL Integration

The schema models OSCAL (Open Security Controls Assessment Language) structures. `project.justfile` contains custom `just` recipes for validating OSCAL catalogs, profiles, and component definitions beyond standard pytest.

### Documentation

Built with MkDocs Material and auto-deployed to GitHub Pages on pushes to `main`. Run `just testdoc` to preview locally.

## Key Configuration Files

- `config.yaml` — LinkML generator settings (controls what artifacts are generated and how)
- `config.public.mk` — Environment variables used by the build system
- `pyproject.toml` — Python metadata, dependency versions, ruff/pytest/codespell configuration
- `.yamllint.yaml` — YAML linting rules applied to schema files and test data
